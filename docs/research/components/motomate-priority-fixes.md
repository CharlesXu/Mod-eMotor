# Mod-eMotor 优先修复规格

参考站：

- 桌面版：<http://motomate.cn:58183/tools/PC/#/>
- 手机版：<http://motomate.cn:58183/tools/M/#/>（手机 User-Agent）

本规格只借鉴已观察到的交互模式与信息架构，不复制参考站源码或第三方视觉素材。

## 1. 原车对比

- “原车”一列必须来自当前所选车型的 `defaultValues`。
- 若车型具备几何校准 profile，原车与当前值必须经过同一套 profile 对齐逻辑。
- 当前值等于车型默认值时，五项变化量均为 0；切换车型后基准随车型改变。

## 2. 数值边界

- 所有数值参数共用一份约束表，作为输入框 `min`、`max`、步进按钮和持久化数据清洗的唯一来源。
- 输入框、加减按钮和加载的本地数据都必须在写入状态前截断到相同边界。
- 边界与 `simulationGeometry.ts` 当前实际采用的范围保持一致，避免界面显示值与几何值不一致。

## 3. 自动化测试

- 增加领域单元测试，覆盖车型基准、参数上下界、非法值回退和边界截断。
- `npm test` 运行全部自动化测试；`npm run check` 必须包含测试。
- 保留现有脱敏、安全、lint、类型和生产构建门禁。

## 4. 响应式布局

- 桌面端保留左侧参数栏与右侧画布。
- 低于 900px 时采用手机版的信息密度：画布占满视口，参数区收纳到下方，模式与显示工具靠边固定，不依赖页面横向滚动。
- 车型选择页在窄屏下使用左侧纵向品牌栏、可滚动的车型卡片区和底部载入按钮；不简单缩放 PC 页面。
- 画布内部按容器缩放，标题、比较表和视图工具不得溢出屏幕。

## 5. 配件与鸣谢

- 选择页和编辑器右上角都提供“配件”和“鸣谢”入口。
- 配件面板列出项目已经支持的加装与相关机械分类，不冒充资源下载仓库。
- 鸣谢面板第一位为 SenZQ，并明确其电鸡模拟器网站是重要参考与资源来源；同时链接第三方声明。
- 面板为可访问对话框，支持显式关闭按钮、遮罩关闭和 Escape 关闭。

## 6. 供应链

- 在不引入破坏性框架升级的前提下更新可安全修复的间接依赖，并记录仍无法消除的告警。
- GitHub Actions 的第三方 Action 使用完整 40 位 commit SHA，并在行尾保留版本标签注释。

## 验收

- 自动化测试、脱敏扫描、依赖审计、lint、类型检查和生产构建全部通过。
- 390px、768px、1440px 三种视口完成本地截图检查，无页面级横向滚动。
- README 末尾是鸣谢章节，SenZQ 位列第一。
