# 配件图片与线条图库规格

## 概览

- 参考站：`http://motomate.cn:58183/tools/PC/#/`
- 目标组件：`src/components/motomate/AccessoryLibrary.tsx`
- 数据文件：`src/data/motomate-accessories.json`、`src/data/motomate-accessories.ts`
- 资源目录：`public/motomate/accessories/`
- 交互模型：点击切换图库、分类和选中状态；搜索即时过滤；图片原生懒加载

## 原站数据范围

| 图库 | 记录数 | 不重复图片数 | 分类 |
| --- | ---: | ---: | --- |
| 配件图片 | 701 | 694 | 前减震、后减震、轮毂、电机、平叉、卡钳、碟片、前挡泥板、后挡泥板、轮胎 |
| 线条图 | 41 | 40 | 平叉、电机、轮毂 |

每条记录完整保留原站的 `name`、`brand`、`type`、`size`、`position`、`position_y`、`concise`、`describe` 和两张图片路径。`picsrc1` 与 `picsrc2` 相同时只展示一张；不同时在同一卡片中并排展示两张。

## DOM 结构

1. 标题栏：标题、保存、返回。
2. 图库切换：`配件图片（701）`、`线条图（41）`。
3. 分类导航：根据当前图库动态展示有数据的分类。
4. 搜索区：按名称、品牌、规格和描述过滤。
5. 卡片网格：图片预览、名称、品牌、规格、选择开关。
6. 空状态与保存提示。

## 状态与行为

- 切换图库后，如果当前分类在新图库不存在，自动切换到新图库的第一个可用分类。
- 每个“图库 + 分类”可独立选择一个配件，再次点击取消。
- 本地保存使用版本化键，读取时只接受当前完整清单中存在的 ID。
- 搜索不改变当前选择，清空搜索后恢复完整分类列表。
- 图片加载失败时显示明确的“图片加载失败”状态，不回退为空白占位。
- 图片使用本地路径，运行时不依赖参考站网络。

## 响应式行为

- 桌面：自适应多列卡片网格，图片预览区高度约 132px。
- 平板：保持至少两列，导航横向滚动。
- 手机（390px）：单列卡片，图库和分类均可横向滚动，图片预览完整可见。

## 验收标准

1. 本地数据为 701 条配件图片和 41 条线稿，记录 ID 唯一。
2. 734 个不重复本地图片文件全部存在且为有效图片。
3. 十个配件图片分类与三个线稿分类均可进入，并显示真实图片而非文字占位。
4. 搜索、选择、取消、保存、返回和 Escape 关闭继续可用。
5. 桌面和 390px 手机视口中卡片图片不溢出、不遮挡文字。
6. 测试、lint、类型检查、生产构建和安全检查全部通过。
